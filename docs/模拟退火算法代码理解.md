# 模拟退火算法代码理解 (`code/drone_delivery_sa.py`)

> **注意**：本文档描述 v2.0 旧版代码（`code/drone_delivery_sa.py`），使用抽象权重成本模型（w1, w2, 惩罚系数M）。
> v3.0 新版代码位于 `src/algorithms/sa.py`，使用统一人民币成本模型。旧代码保留在 `code/` 作为参考。
> 项目最新架构详见 `docs/总体代码理解.md`。

---

## 一、整体结构

```
run_sa(clients, num_drones)  ← 统一入口（与 run_genetic 接口一致）
    │
    ├─► simulated_annealing()  ← 核心：多轮重启式模拟退火
    │       │
    │       ├─► evaluate_solution()  ← 适应度 = 总飞行距离 + 约束惩罚
    │       │       └─► decode_individual()  ← 复用遗传算法的解码器
    │       │
    │       └─► generate_neighbor()  ← 邻域搜索（Swap / Reverse / Insert）
    │
    ├─► print_results()            ← 控制台 + 文本文件输出
    ├─► plot_routes()              ← 总览路线图 + 统计面板
    ├─► plot_annealing_curve()     ← 每轮重启的退火轨迹（温度+成本）
    └─► plot_convergence_summary() ← 所有重启的收敛曲线叠加对比
```

## 二、超参数（第 29–35 行）

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `SA_T0` | 500.0 | 初始温度 —— 高温时接受劣解的概率大，保证搜索广度 |
| `SA_T_MIN` | 0.01 | 终止温度 —— 低温时几乎只接受改进解，收敛到局部最优 |
| `SA_ALPHA` | 0.98 | 降温速率 —— `T = T × 0.98`，值越接近 1 降温越慢、搜索越充分 |
| `SA_ITER_PER_T` | 50 | 每个温度下的迭代次数 —— 在同一个温度平台上的搜索步数 |
| `SA_MAX_RESTARTS` | 3（TW模式自动+2→5） | 重启动次数 —— 部分 SA 可能陷入较差局部最优，多轮独立搜索取最优 |
| `SA_ITER_PER_T` | 50（TW模式自动+30→80） | 每个温度下的迭代次数 —— 时间窗模式下搜索空间更复杂，自动增加 |
| `PENALTY_WEIGHT` | 1e4 | 约束违反惩罚系数 —— 与遗传算法一致，确保不可行解被淘汰 |

**温度下降过程示例**：T 从 500 开始，每轮 ×0.98，经 26800 次迭代后降至 0.01 以下（约 `500 × 0.98^26800 ≈ 0.01`）。

## 三、核心函数

### 3.1 `evaluate_solution()` — 适应度函数（第 52–59 行）

```
输入：
  individual   ← list[int]，客户索引 0..n-1 的一个排列，如 [5,12,0,3,7,...]
  clients      ← shape=(n,3) 客户数据 [x, y, weight]
  dist_matrix  ← shape=(n+1, n+1) 距离矩阵，第0行/列为配送中心
  num_drones   ← int，无人机数量
  time_windows ← 可选 (ready_times, due_times)，None 表示不启用时间窗

输出：
  (cost, routes) ← (float, list[dict])
    cost   = 总飞行距离 + 约束惩罚（含超重/超距/迟到惩罚，值越小越好）
    routes = 每条路线是一个 dict：drone_id, depart_time, arrive_time,
             route, load, distance, deliveries
```

**逻辑**：调用 `decode_individual`（复用遗传算法的解码器）将排列解码为满足载重/里程/时间窗约束的路线合集，返回总距离 + 总惩罚值。

---

### 3.2 `generate_neighbor()` — 邻域算子（第 62–88 行）

```
输入：
  individual ← list[int]，当前解（排列）

输出：
  neighbor ← list[int]，邻域解（仍是合法排列）
```

**三种算子**（随机概率选择）：

| 概率 | 算子 | 操作 | 图示 |
|------|------|------|------|
| 40% | **Swap** 交换 | 随机交换两个位置的值 | `[A,B,C,D]` → `[A,D,C,B]` |
| 30% | **Reverse** 反转 | 反转一段随机子序列（2-opt 思想） | `[A,B,C,D]` → `[A,D,C,B]` |
| 30% | **Insert** 插入 | 将一个元素从位置 i 拔出，插入到位置 j | `[A,B,C,D]` → `[A,C,B,D]` |

三种算子都保证输出仍是合法排列（不缺不重），不需要额外修复。

### 3.3 `simulated_annealing()` — 核心退火算法（第 91–165 行）

这是整个文件最核心的函数。

```
输入：
  clients      ← shape=(n,3) 客户数据
  dist_matrix  ← shape=(n+1, n+1) 距离矩阵
  num_drones   ← int，机队规模
  t0, t_min, alpha, iter_per_t ← 退火超参数（可选，默认用模块级常量）
  time_windows ← 可选 (ready_times, due_times)，None 表示不启用时间窗

输出：
  best_overall        ← list[int]，全局最优排列
  best_routes_overall ← list[dict]，全局最优路线
  best_cost_overall   ← float，全局最优成本
  all_cost_history    ← list[(cost_history, temp_history)]，每个重启的搜索轨迹
```

**逐段逻辑（第 103–165 行）**：

```
第 104 行   设置随机种子（保证可复现）
第 107-111  初始化"全局最优"追踪变量为 inf

第 113 行   外层循环：SA_MAX_RESTARTS 次重启
第 115-116   随机生成初始解（0..n-1 的随机排列）
第 117-119   评估初始解的适应度

第 121-123   初始化"本轮最优" = 初始解
第 125-128   初始化本轮搜索轨迹记录
第 127       T = t0（初始温度）
第 128       total_iter = 0（总迭代计数器）

第 130 行    while T > t_min:              ← 温度降到阈值以下时停止
第 131 行      for _ in range(iter_per_t):  ← 每个温度平台迭代 iter_per_t 次
第 132-135      生成邻居 + 评估邻居成本
第 136          delta = 邻居成本 - 当前成本

第 138 行        if delta < 0 or random() < exp(-delta/T):
                    ← Metropolis 准则：
                      - 邻居更优 → 一定接受
                      - 邻居更差 → 以 exp(-ΔE/T) 的概率接受
                      （T 高时容易接受劣解，T 低时几乎只接受优解）

第 139-141        更新当前解
第 143-146        如果比本轮最优更好 → 更新本轮最优

第 148-150      记录本轮最优成本和当前温度到历史中

第 152 行      T *= alpha  ← 降温

第 157-160   如果本轮最优 优于 全局最优 → 更新全局最优
第 162        保存本轮搜索轨迹

第 165 行   返回全局最优
```

**Metropolis 准则解释**：

`exp(-delta / T)` 的含义：
- delta = 100（邻居比当前差 100），T = 500 → 接受概率 `exp(-100/500)` ≈ 81.9%
- delta = 100（邻居比当前差 100），T = 10 → 接受概率 `exp(-100/10)` ≈ 0.0045%
- 高温时"大胆跳跃"逃离局部最优，低温时"精细打磨"逼近最优

## 六、统一入口

### 6.1 `run_sa()` — 管线入口（第 486–521 行）

```
输入：
  num_drones=5    ← 无人机数量
  clients=None    ← 客户数据（None 则自动生成并清洗）
  output_dir      ← 输出目录
  time_windows=None ← 可选时间窗 (ready_times, due_times)

输出：
  {
    'clients':           客户数据数组,
    'routes':            最优路线列表,
    'dist_matrix':       距离矩阵,
    'total_distance':    总飞行距离,
    'total_trips':       总趟次,
    'total_deliveries':  总配送次数,
    'all_cost_history':  退火搜索轨迹,
  }
```

**与 `run_genetic()` 接口完全一致**，在 `main.py` 中可直接互换调用。

### 6.2 `main()` — 独立运行入口（第 524–565 行）

支持命令行参数：`--num-drones`（机队规模）、`--t0`（初始温度）、`--alpha`（降温速率）、`--output-dir`（输出目录）。

7 个步骤：数据生成 → 清洗 → 距离矩阵 → SA 优化 → 文本输出 → 路线可视化 → 退火过程可视化。

## 七、与遗传算法的关系

| 层级 | 关系 |
|------|------|
| 编码 | **相同**：都是客户索引 0..n-1 的排列 |
| 解码 | **复用**：`decode_individual()` 和 `FleetScheduler` 直接从 `drone_delivery_genetic` 导入，含 EDD 排序和时间窗感知归组 |
| 评估 | **相同逻辑**：总距离 + 约束惩罚（含时间窗迟到惩罚），最小化 |
| 搜索策略 | **不同**：SA 用 Metropolis 准则 + 温度调度；GA 用种群进化（选择/交叉/变异/精英保留） |
| 优势 | SA 实现更简洁（无需 DEAP 库），调参更直观（温度/降温速率 vs 种群/交叉/变异）；单次运行更快，适合快速迭代 |
