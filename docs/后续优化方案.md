## 二、代码层优化（可选，工时允许时做）

### 2.1 低投入高回报（推荐）

**统一可视化主题**
- 将暗色配色常量提取到 `common.py`
- 贪心算法可视化改为暗色主题，与遗传/退火风格一致
- 工作量：约30分钟

**完善注释**
- 为每个公开函数补一行 docstring
- 标注关键数值的含义（如惩罚系数 1e4 的选择依据）
- 工作量：约30分钟

**消除 print 散落**
- 将 `print()` 替换为 `logging` 模块，支持静默模式
- 工作量：约1小时

### 2.2 中等投入（可选加分）

**增加 Streamlit 交互界面**
```python
# 核心功能：
# 1. 侧边栏：选择算法、调节参数（无人机数量、温度、种群大小等）
# 2. 主区域：路径图 + 指标对比表 + 收敛曲线
# 3. 支持一键切换算法重新运行
```
- 新增文件：`code/app.py`（约200行）
- 依赖添加：`streamlit` 到 `requirements.txt`
- 工作量：约1天
- 收益：加分项，答辩时现场演示效果好

**增加真实场景约束（选1-2个）**
- **时间窗约束**：每个客户有指定配送时间窗口，早到/晚到产生惩罚
- **续航衰减**：载重越大能耗越高，实际里程随载重递减
- **禁飞区**：地图上添加多边形禁飞区，路径需绕行
- **多配送中心**：支持多个配送中心，无人机从最近中心出发
- 工作量：每个约束约0.5-1天

### 2.3 低优先级

- 单元测试：`pytest` 覆盖 decode/evaluate 等核心函数
- Type Hints：为关键函数添加类型标注
- 配置文件：将超参数从硬编码改为 YAML/JSON 配置文件

## 三、中期报告的不足之处（需在技术报告中改进）

对比中期报告内容，最终技术报告需要提升的地方：

| 中期问题 | 最终报告改进方向 |
|---------|----------------|
| 只描述了贪心算法 | 重点写遗传算法和模拟退火的原理、设计和对比 |
| 数据处理较简单 | 可以补充数据分析（客户分布热力图、距离分布直方图） |
| 缺少定量对比 | 三种算法详细对比表（距离、趟次、收敛速度、运行时间） |
| 缺少算法原理深度 | 用伪代码/公式说明编码方式、选择/交叉/变异算子、Metropolis准则 |
| 缺少参数分析 | 遗传算法超参数网格搜索结果，模拟退火降温曲线分析 |
