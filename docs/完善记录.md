### 新增 `code/main.py` — 多种子评估功能

**目的**：解决原项目仅在单一样本（种子42）上评估的问题，使算法对比结论具有统计意义。

**新增功能**：
- 新增 `--seeds N` 命令行参数，支持在 N 个不同随机种子上独立运行算法
- 每个种子生成不同的客户分布数据，各算法独立求解
- 输出均值±标准差汇总表（总距离、趟次、配送次数）
- 各次详细结果逐行展示（便于观察波动范围）
- 汇总自动保存为 `multi_seed_summary_*.txt` 到 outputs 目录
- 最后一个种子的完整可视化（图表+文本）照常输出
- 多种子模式下自动启用 Agg 后端（不弹窗）

**使用方式**：
```bash
python main.py --algo all --seeds 5     # 5个种子上三种算法对比
python main.py --algo both --seeds 10   # 10个种子上贪心vs遗传
```

**新增函数**：
- `print_multi_seed_summary()` — 打印多种子汇总统计
- `run_multi_seed_evaluation()` — 多种子评估主循环

**设计考量**：
- 非末种子仅收集指标不保存文件，避免输出目录膨胀
- 末种子调用原有的 `run_*` 函数生成完整可视化输出
- 与单次运行模式共享命令行参数（`--algo`、`--num-drones`、`--output-dir`）

### 新增 `code/app.py` — Streamlit 交互式 Web 界面

**目的**：为答辩汇报提供可现场演示的交互式界面（加分项），用户可通过侧边栏自由切换算法、调参并实时查看结果。

**功能概览**：
- **算法选择**：贪心/遗传/退火/全部对比 四种模式
- **参数调节**：客户数量(10-50)、无人机数量(3-20)、随机种子
- **遗传算法超参**：种群规模、迭代代数、交叉/变异概率（可折叠面板）
- **模拟退火超参**：初始温度、降温速率、重启次数（可折叠面板）
- **全部对比模式**：
  - 三算法并排路径对比图
  - 指标卡片（总距离 + 趟次）
  - 对比表格（含 vs贪心提升百分比）
  - 总距离柱状图
  - 遗传算法进化曲线
  - 模拟退火搜索轨迹（含温度双Y轴）
- **单算法模式**：路径图 + 对应收敛曲线
- 暗色主题与代码可视化风格统一

**技术要点**：
- `matplotlib.use('Agg')` 在导入 pyplot 前设置，避免 plt.show() 弹窗
- 通过修改模块级常量（`drone_delivery_genetic.GA_POP_SIZE` 等）实现实时调参
- `st.session_state` 保存计算结果，参数变更不触发重复计算
- `st.spinner` 提供运行中的加载反馈

**使用方式**：
```bash
cd code/
pip install streamlit pandas
streamlit run app.py
```

**运行方式说明**：在终端执行上述命令后，浏览器会自动打开 `http://localhost:8501`，即可在 Web 界面中操作。

### 1. 新建 `code/common.py` — 共享工具模块

- `compute_distance_matrix()` — 距离矩阵计算

### 2. 修复 `code/drone_delivery.py` — 贪心算法改进

**修复的死循环 bug**：
- 原代码：当所有剩余客户都无法被装入路线时，内层 while 循环 break 但 `current_route` 为空，外循环不会移除任何未分配客户，导致无限循环
- 修复：当 `current_route` 为空时，强制逐个分配剩余客户并记录超限警告，避免死循环

**贪心策略改进**：
- 原代码：选最近客户，装不下就直接放弃整条路线（`break`）
- 改进：将未分配客户按距离排序后逐个尝试，第一个装不下则试第二个、第三个...直到全部试完才开启新路线

**其他改进**：
- 改为 `from common import ...` 消除重复代码
- 添加 `--output-dir` 命令行参数，输出到 `outputs/` 目录
- 添加 `run_greedy()` 函数，方便被 `main.py` 调用

### 3. 修复 `code/drone_delivery_genetic.py` — 遗传算法修复

**修复的死循环 bug**：
- 原代码：`decode_individual` 中当 `route` 为空时执行 `else` 分支（强制装入），但 `i += 1` 后回到外层循环，如果还有客户，可能再次进入同样的死循环逻辑
- 修复：当 `route` 为空时强制装入第一个客户后 `continue` 或直接 `break`，同时将单客户超限的分支逻辑在 `route` 为空时就处理好

**其他改进**：
- 改为 `from common import ...` 消除重复代码
- 添加 `--output-dir` 和 `--num-drones` 命令行参数
- 添加 `run_genetic()` 函数，方便被 `main.py` 调用
