### 摘要（~250字）

**写作要点**：一句话背景 → 问题定义 → 方法概述 → 核心结果 → 结论。

**可直接使用以下草稿**：

> 本文研究无人机集群配送路径规划问题。针对多无人机、多客户点、多时间窗的低空物流配送场景，以最小化统一人民币总成本为优化目标，在载重、航程和时间窗约束下设计并实现了四种求解算法。贪心算法基于 Best-Fit 解码和距离排序构建快速基线；模拟退火采用五种邻域操作和自适应冷却策略进行迭代优化；遗传算法基于排列编码和精英保留策略进行种群进化搜索；随机搜索作为对比基线。在标准场景（30客户/5无人机）下进行实验，结果表明：模拟退火取得最优成本 1,443 元，较贪心基线（2,446 元）降低 41%，且实现零延迟；遗传算法次之（1,628 元，降低 33%）；随机搜索成本高达 21,107 元且违反硬约束，验证了优化算法的有效性。本文的算法框架和对比分析方法可为低空物流路径优化提供参考。

### 2. 问题描述与建模（~500字）

**2.1 场景描述（~150字）**

要点：
- 仓库位于 50×50 km 区域中心 (25, 25)
- M 架同型号无人机：载重 C=20kg，速度 v=40km/h，最大航程 D_max=150km
- N 个客户，各有坐标、需求量（1–8kg）、类型、时间窗、服务时间
- 每架无人机只执行单趟配送（总需求 ≤ 0.85 × M × C）
- 目标：决定每架无人机访问哪些客户及顺序，最小化总成本

**2.2 数学建模（~250字）**

**决策变量**：
```
x_{ijk} ∈ {0, 1} — 无人机 k 是否从客户 i 飞到客户 j
```

**距离计算**：
```
dist_{ij} = β × √((x_i − x_j)² + (y_i − y_j)²),  β = 1.3（道路曲折系数）
```

**时间窗处理**（核心建模细节，需写清楚）：
```
arrival_time  = 无人机到达客户点的时间
start_service = max(arrival_time, e)      # 不早于时间窗开始
waiting_time  = max(0, e − arrival_time)  # 提前到达→等待
delay_time    = max(0, arrival_time − l)  # 晚到→延迟

时间链: depart_i = start_i + service_i → arrive_{i+1} = depart_i + dist/v
```

**目标函数（统一人民币）**：

```
Minimize: TotalCost = 0.8 × TotalDistance(km)
                    + 1.0 × TotalOperationTime(min)
                    + 20.0 × TotalDelay(min)
                    + 500.0 × TotalOverload(kg)
                    + 300.0 × TotalExcessRange(km)
```

| 成本项 | 系数 | 物理含义 |
|--------|------|----------|
| 飞行成本 | 0.8 元/km | 电费 + 维护 + 折旧 |
| 运营时间成本 | 1.0 元/min | 飞行+等待+服务时间 |
| 延迟罚金 | 20.0 元/min | 超时惩罚（模拟客户投诉成本） |
| 超重罚金 | 500.0 元/kg | 超载惩罚 |
| 超航程罚金 | 300.0 元/km | 超航程惩罚 |

**约束条件**（公式形式，建议 LaTeX）：

| 约束 | 公式 | 说明 |
|------|------|------|
| 容量约束 | Σᵢ demandᵢ × Σⱼ x_{ijk} ≤ C, ∀k | 硬约束 |
| 航程约束 | Σᵢ Σⱼ dist_{ij} × x_{ijk} ≤ D_max, ∀k | 硬约束 |
| 访问唯一性 | Σ_k Σⱼ x_{ijk} = 1, ∀i | 硬约束 |
| 流守恒 | Σⱼ x_{jik} = Σⱼ x_{ijk}, ∀i,k | 硬约束 |
| 仓库约束 | Σⱼ x_{0jk} = 1, Σᵢ x_{i0k} = 1, ∀k | 硬约束 |
| 时间窗（软） | start_service_i ∈ [e_i, l_i] | 迟到罚金，早到计运营成本 |

**2.3 客户类型体系（~100字）**

| 类型 | 占比 | 时间窗时段 | 窗宽 | 设计理由 |
|------|------|-----------|------|----------|
| 紧急 urgent | 20% | 08:00–09:00 | 30–45min | 生鲜/药品/早餐 |
| 普通 normal | 60% | 09:00–12:00 | 60–120min | 常规快递 |
| 宽松 relaxed | 20% | 10:00–16:00 | 120–240min | 日用品/大包裹 |

错峰设计使得时间窗具有真正的调度意义——算法需要权衡"先等紧急客户开放"还是"先飞远路送宽松客户"。

### 3. 算法设计（~1800字，报告核心）

**3.1 编码方案与解码过程（~300字）**

这是三种元启发式算法的**共享基础**，务必在 3.1 节统一说明：

> 将 N 个客户索引编码为一个排列 π = [c₁, c₂, ..., c_N]。解码时按排列顺序，使用 Best-Fit 策略将客户依次分配给 M 架无人机：对每个客户，选择剩余容量最充足且能装入的无人机；若无法装入，则放入负载最轻的无人机（超载由罚金处理）。分配完成后，每条路线内按截止时间（EDD）升序排序以减少延迟。解码完成后自动过滤空路线。

- 这种编码天然保证每个客户恰好被访问一次
- 排列编码的搜索空间大小为 N!，远小于直接编码路线的组合空间

**建议配图**：一个编码→解码流程简图

**3.2 贪心算法基线（~300字）**

**最近优先策略**（`solve_greedy`）：
```
order = sorted(range(N), key=lambda i: dist[depot, i])
routes = _decode_greedy(problem, order)
```

**紧急优先策略**（`solve_greedy_urgent`）：
```
order = sorted(range(N), key=lambda i: (type_rank[ctype], dist[depot, i]))
routes = _decode_greedy(problem, order)
```

**_decode_greedy 核心逻辑**（用伪代码或简洁文字描述）：
1. 初始化 M 条空路线
2. 按 order 顺序遍历每个客户 ci：
   - Best-Fit: 在所有无人机中找能装入 ci 且剩余容量最大的
   - 若无架能装入 → 放入负载最轻的（超载由罚金覆盖）
3. 各路线内按 EDD 升序排序（截止早的先送）

**优化点分析**：
- Best-Fit（非 First-Fit）：减少容量碎片化，提高无人机利用率
- EDD 趟内排序：降低时间窗违反风险
- 两种排序策略可适应不同业务场景

复杂度 O(N²)，耗时 < 0.01s，产出 2,446 元的初始可行解，同时作为 SA 的初始解。

**3.3 模拟退火算法（~600字）**

**基本思想**：
源自冶金学中金属退火过程——将金属加热至高温后缓慢冷却，原子在高温时剧烈运动（探索），降温后趋于有序排列（收敛）。映射到优化问题：温度 T 控制搜索的随机程度。

**Metropolis 接受准则**（加框公式）：

```
P(接受新解) = 1                        , 若 ΔE ≤ 0（新解更优）
            = exp(−ΔE / T)            , 若 ΔE > 0（新解更差）
```

其中 ΔE = Cost(新解) − Cost(当前解)，T 为当前温度。

**温度的作用机制**：
- T 高（如 1000）时，exp(−ΔE/T) ≈ 1 → 即使差很多也大概率接受 → 全局探索
- T 低（如 0.1）时，exp(−ΔE/T) ≈ 0 → 几乎不接受差解 → 局部精调

**五种邻域操作**（每次迭代随机选取一种）：

| 操作 | 作用域 | 描述 |
|------|--------|------|
| 2-opt | 路内 | 逆转路径中的一段子序列 |
| Swap | 路内 | 交换同路线内两个客户位置 |
| Relocate | 跨机 | 将一个客户从 A 路线移到 B 路线 |
| Exchange | 跨机 | 交换 A、B 路线各一个客户 |
| Or-opt | 跨机 | 将连续 2-3 个客户整体迁移到另一路线 |

**冷却策略**：
- 初始温度 T₀ = 1000，终止温度 T_min = 0.01
- 指数降温：T ← T × α，α = 0.98
- 每个温度下迭代 100 次
- **自适应加速**：连续 50 步无改善 → α ← α × 0.8，加速降温跳出无效搜索

**初始解**：贪心算法结果。

**伪代码**（建议放简化版）：
```
T ← T₀
当前解 ← 贪心初始解
while T > T_min:
    for i = 1 to 100:
        新解 ← 随机邻域操作(当前解)
        ΔE ← Cost(新解) − Cost(当前解)
        if ΔE < 0 or random() < exp(−ΔE/T):
            当前解 ← 新解
    T ← T × α
    若连续50步无改善: α ← α × 0.8
```

**3.4 遗传算法（~400字）**

**编码方案**：客户排列编码（同 3.1 节），解码时用 Best-Fit 贪心分割为多条路线。

**适应度函数**：直接使用目标函数 TotalCost（最小化问题，DEAP 中 weights = (−1.0,)）。

**遗传算子**：

| 算子 | 方法 | 概率 | 作用 |
|------|------|------|------|
| 选择 | 锦标赛选择 (size=3) | — | 优胜劣汰，随机取3个选最优 |
| 交叉 | 有序交叉 OX | 0.8 | 保留父代排列中的相对顺序 |
| 变异 | 逆转变异 | 0.075 | 逆转一段子排列 |
| 变异 | 交换变异 | 0.075 | 交换排列中两个位置 |

**精英保留与早停**：
- 每代保留最优 5 个个体直接进入下一代
- 若连续 50 代最优个体无改善 → 提前终止

**超参数配置**：

| 参数 | 值 |
|------|-----|
| 种群大小 | 100 |
| 最大代数 | 200 |
| 交叉概率 | 0.8 |
| 变异概率 | 0.15（两类各半） |
| 锦标赛规模 | 3 |
| 精英数量 | 5 |
| 早停代数 | 50 |

**实现**：基于 DEAP 框架（`deap.algorithms.eaSimple` + 自定义算子）。

**3.5 随机搜索基线（~200字）**

- 生成 10,000 个随机客户排列
- 用相同的 Best-Fit 解码器评估每条排列的总成本

- 预期结果：随机搜索成本远高于优化算法，且大概率违反约束

**4.3 收敛性分析（~250字）**

**SA 收敛特征**（观察 `fig5_convergence.png`）：
- 初始阶段（迭代 < 500）：成本从 2,446 元快速下降至约 1,800 元，高温段接受大量差解，广搜

**关键洞察**：SA 单次运行即可收敛到 1,443 元，无需多次重启；GA 的收敛速度更快但最终解略差于 SA。

### 5. 讨论（~300字）

**5.1 算法选择的工程建议（~150字）**

| 场景 | 推荐算法 | 理由 |
|------|----------|------|
| 在线实时调度 | 贪心 | <0.01s 出解，延迟可接受 |
| 离线配送规划 | **模拟退火** | 成本最优，零延迟，实现简单 |
| 超大规模（>100客户） | 遗传算法 | 可并行化，扩展性好 |
| 快速原型验证 | 贪心 | 无需调参，代码量小 |

**5.2 局限性（~100字）**

- 仿真数据与真实场景有差距（未考虑天气、地形、禁飞区、3D路径）
- 配送为静态规划，未考虑动态插入新订单
- 未考虑无人机充电/换电
- 单趟模式假设每架无人机只飞一趟，多趟模式需要扩展
- SA 和 GA 的超参数依赖经验调参，不同场景可能需要重新调优

**结论（~150字）**：
