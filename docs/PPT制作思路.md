**左侧文字**：

> **背景**：低空物流是人工智能优化的典型应用场景。多无人机协同配送需要解决"谁飞哪条路线"的组合优化问题（VRP变种）。

> **问题定义**：
> - 配送中心位于坐标 (50, 50)

>   - 单架无人机最大载重 ≤ 20 kg
>   - 单趟飞行最大里程 ≤ 200 单位
>   - 所有客户必须被服务
>   - （可选）每个客户有配送时间窗口

**右侧图片**：客户点分布散点图

- 用虚线大圆标注"最大里程范围"（半径 100，即 200/2）
- X/Y 轴范围 [0, 100]

**口述要点**（演讲者备注）：
> "我们的问题是：1个配送中心、20个客户、N架无人机，目标是总飞行距离最短。约束是每架飞机载重不超过20公斤、单趟不超过200单位。这本质上是一个带容量和距离约束的车辆路径问题。"

### 第5页：算法总览（三种算法对比框架）

**布局**：上半部分为对比表格，下半部分为统一流程图

**上半 — 对比表格**：

| | 贪心算法 | 遗传算法 | 模拟退火 |
|------|---------|---------|---------|
| **类型** | 构造启发式 | 元启发式（进化计算） | 元启发式（物理模拟） |
| **核心思想** | 每步选最近可行客户 | 种群进化，适者生存 | 温度递减，概率接受劣解 |

**下半 — 统一流程图**（横向箭头 + 图标）：

  ↑ 三种算法都在搜索最优的"客户访问排列"                              ↑

### 第6页：统一编码方案（排列编码 + 解码）

**布局**：左编码 → 中解码 → 右评估，三步横向流程

**左侧 — 编码**（占 25%）：
```
个体（排列）:
[3, 7, 0, 12, 5, 18, 9, ...]
 ↑  ↑  ↑
 │  │  └─ 第3个被访问的客户是0号
 │  └──── 第2个被访问的客户是7号
 └─────── 第1个被访问的客户是3号

长度 = n（客户数）
每个数字出现且仅出现一次
```

**中间 — 解码**（占 45%）：
```
解码流程（贪心打包）：

depart_time = 最早空闲飞机的就绪时刻
current_time = depart_time

按排列顺序逐个尝试装入当前趟次：
┌─────────────────────────────────────┐
│ 客户3: 载重 ≤ 20? ✓  里程 ≤ 200? ✓  │ → 装入趟次1
│ 客户7: 载重 ≤ 20? ✓  里程 ≤ 200? ✓  │ → 装入趟次1
│ 客户0: 载重 ≤ 20? ✗                 │ → 结束趟次1，开启趟次2
│ 客户0: 载重 ≤ 20? ✓  里程 ≤ 200? ✓  │ → 装入趟次2
│ ...                                  │
└─────────────────────────────────────┘

结果：
  趟次1: [3, 7]     载重8kg  里程120
  趟次2: [0, 12, 5] 载重11kg 里程165
  趟次3: [18, 9, ...]
```

**右侧 — 评估**（占 30%）：
```
适应度 = 所有路线总飞行距离 + 约束惩罚

惩罚计算：
  超重惩罚 = 10000 × max(0, 实际载重 - 20)
  超距惩罚 = 10000 × max(0, 实际里程 - 200)

适应度越小越好（最小化问题）
```

**设计提示**：用不同颜色标注不同的趟次。三个部分之间用大箭头连接。

**口述要点**：
> "排列编码是整个系统的核心。为什么用排列？因为它天然保证每个客户只访问一次，不需要复杂的约束处理。解码器按排列顺序'贪婪打包'：前面的客户填满一趟就自动开下一趟。适应度等于总距离加约束惩罚，惩罚系数1e4确保不可行解被淘汰。"

### 第7页：贪心算法（基线方法）

**布局**：左文右图（40:60）

**左侧文字**：
> **策略**：改进的最近邻贪心
> 1. 从配送中心出发
> 2. 计算所有未分配客户的有效成本：
>    - 无时间窗：`cost = 距离`
>    - 有时间窗：`cost = 距离 + (迟到×3.0 + 等待×0.1) × 速度`
> 3. 按有效成本排序，逐个尝试装入
> 4. 第一个满足载重+里程约束的客户入选
> 5. 重复直到无人可装 → 返回配送中心 → 开启新趟次
> 6. 兜底：若剩余客户单独都无法装入，强制逐人分配（带警告）

> **复杂度**：O(n²)，每次选择需扫描剩余客户

> **典型结果**（种子=42）：总距离 716.65，5 趟，20 客户全部配送

**右侧图片**：贪心算法路径图（暗色主题）
- 配送中心：红色光晕五角星
- 各趟次用不同颜色区分的箭头路线
- 路线中心标注载重/里程
- 图例按趟次着色

**口述要点**：
> "贪心算法是性能基线。它每次选择'看起来最好'的客户——距离最近且能装得下的。改进版从不装不下就直接放弃路线，而是尝试下一个次近客户。最终得到总距离717、5趟次，作为后续两种算法的对比基准。"

**上半 — 进化流程**（横向）：
```
[初始种群 200个体] → [评估适应度] → ┌→ [选择:锦标赛] → [交叉:有序交叉] → [变异:洗牌索引] → [精英保留10个]┐

```

**下半 — 四张算子卡片**（2×2 网格）：

| **选择：锦标赛选择** | **交叉：有序交叉 (cxOrdered)** |
|---------------------|------------------------------|
| • 随机取 5 个个体 | • 相邻两个体配对 |
| • 保留适应度最优的 | • 80% 概率执行交叉 |
| • 重复 190 次（200-10精英位） | • 从父代A取一段，父代B补全剩余 |
| • 优者多选，劣者也有机会 | • 保证子代仍是合法排列 |

| **变异：索引洗牌 (mutShuffleIndexes)** | **精英保留** |
|--------------------------------------|-------------|
| • 每个个体 30% 概率变异 | • 每代进化后保留前 10 名 |
| • 随机洗牌 indpb=2/n≈10% 的位置 | • 直接进入下一代，不经选择/交叉/变异 |
| • 维持种群多样性 | • 保证最优解不会退化 |

**超参数表**（底部横条）：
```
种群=200  |  代数=200  |  交叉率=0.8  |  变异率=0.3  |  锦标赛=5  |  精英=10
（经过网格搜索调优：{100,200,300}×{200,300,500}×交叉×变异，最优组合总距离548.89）
```

**口述要点**：
> "遗传算法模仿自然进化。200个个体组成种群，每个个体是一个排列。每代经过选择（锦标赛）、交叉（PMX风格有序交叉）和变异（洗牌索引），保留精英10个。关键是所有算子都保证子代仍是合法排列。经过网格搜索调优的超参数。"

### 第10页：模拟退火：核心机制

**布局**：左原理 + 右公式，下半为邻域算子

**左侧 — 退火类比**（占 45%）：

| 物理退火 | → | 优化搜索 |
|---------|---|---------|
| 高温：分子剧烈运动 | → | 大概率接受劣解（广域探索） |
| 降温：运动逐渐减弱 | → | 接受劣解概率降低（聚焦精调） |
| 常温：分子趋于稳定 | → | 只接受优解（收敛局部最优） |

**温度下降曲线示意图**（T从500指数衰减到0.01）：
```
T=500 ┤╲
      │ ╲
      │  ╲____
      │       ╲____
T=0.01┤──────────────
      0          ~26800 迭代
```

**右侧 — Metropolis 准则**（占 55%）：

> **核心公式**：
> ```
> 接受概率 P = { 1,                    if ΔE < 0  (邻居更优，一定接受)
>              { exp(-ΔE / T),         if ΔE ≥ 0  (邻居更差，概率接受)
> ```
>
> **数值示例**：
> - ΔE = 100, T = 500 → P = e^(-100/500) ≈ **81.9%** （高温，大胆尝试）
> - ΔE = 100, T = 100 → P = e^(-100/100) ≈ **36.8%** （中温，趋于谨慎）
> - ΔE = 100, T = 10  → P = e^(-100/10)  ≈ **0.005%**（低温，几乎拒绝）

**下半 — 三种邻域算子**（3 列并排）：

| **Swap 交换 (40%)** | **Reverse 反转 (30%)** | **Insert 插入 (30%)** |
|---------------------|----------------------|---------------------|
| 随机交换两个位置 | 反转一段子序列 | 将元素从i位拔出插入到j位 |
| `[A,B,C,D,E]` | `[A,B,C,D,E]` | `[A,B,C,D,E]` |
| `[A,E,C,D,B]` | `[A,D,C,B,E]` | `[A,C,B,D,E]` |

**超参数**（底部横条）：
```
初始温度 T₀=500  |  降温速率 α=0.98  |  终止温度=0.01  |  每温迭代=50次  |  重启次数=3
```

**口述要点**：
> "模拟退火的核心是Metropolis准则：ΔE<0一定接受，ΔE≥0以e^(-ΔE/T)概率接受。温度高时接受劣解的概率大，保证搜索广度；温度低时几乎只接受优解，聚焦局部打磨。三种邻域算子随机选用，覆盖不同的搜索模式。"

### 第12页：时间窗约束（VRPTW扩展）

**布局**：上描述 + 下三列处理方式

**上方 — 问题升级**：

> **从 VRP 到 VRPTW**：每个客户增加了配送时间窗口 `[ready_time, due_time]`
> - **早到**：允许等待至 `ready_time`（无人机在客户处等待）
> - **迟到**：施加惩罚 `10000 × (arrival_time - due_time)`
> - **目标**：在满足时间窗的前提下最小化总飞行距离

**时间窗生成示意图**（横向时间轴）：
```
客户A:  depot──→├────[====时间窗====]────┤
        0      12.3                    27.8    (时间单位)
                 ↑ ready                ↑ due

客户B:  depot──→├──────────[====时间窗====]──────────┤
        0      20.0                                  59.2
```

**下方 — 三种算法的时间窗处理**（3 列）：

| **贪心算法** | **遗传算法** | **模拟退火** |
|-------------|-------------|-------------|
| 候选排序加入时间窗感知权重 | `decode_individual` 追踪 `current_time` | 复用遗传的解码器 |
| `key = 距离 + (迟到×3 + 等待×0.1) × 速度` | `arrival = current_time + 飞行时间` | 与遗传算法完全一致 |
| 优先选择时间窗兼容的客户 | `current_time = max(arrival, ready) + 服务时间` | 惩罚并入适应度 |
| 迟到权重远大于等待权重 | 迟到惩罚 → 适应度 | 优化器自然规避迟到 |

**底部 — 关键说明**：

> - 时间窗是**软约束**：允许迟到，但通过大惩罚让优化器主动规避
> - 向后兼容：`time_windows=None` 时行为与无时间窗完全一致
> - 启用方式：命令行 `--tw` 标志，或 Streamlit 勾选"启用时间窗约束"

**口述要点**：
> "我们进一步将经典VRP扩展为VRPTW——每个客户增加了配送时间窗。这是更贴近真实场景的模型：比如生鲜配送必须在指定时间段内完成。处理方式是软约束+大惩罚，三种算法均已支持。"

## 三、PPT 制作规范

### 视觉风格
- **配色方案**：统一使用暗色主题（`#0d1117` 背景，`#161b22` 面板，与代码可视化一致）
- **强调色**：蓝色 `#58a6ff`（遗传）、绿色 `#3fb950`（退火）、橙色 `#f78166`（高亮数值）
- **字体**：中文用微软雅黑或思源黑体，英文用 Arial 或 Segoe UI
- **字号**：标题 ≥ 32pt，正文 ≥ 20pt，图表标注 ≥ 14pt，表格内容 ≥ 16pt
- **避免大段文字**：用要点列表（每条 ≤ 2 行），关键数字加粗放大

### 通用设计原则
- 每页一个核心信息，不要让观众猜"这页在说什么"
- 图表 > 文字：能用图就不用文字，能用表就不用段落
- 关键数字用大号加粗 + 强调色（如 **532.99**、**+25.6%**）
- 代码片段用等宽字体，深色背景 + 语法高亮
- 保持一致的页边距、标题位置、字体大小
