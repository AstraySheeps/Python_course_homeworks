### 第7页：贪心算法 — 基线构建与源码解读（成员B讲）

- **标题**: 贪心算法：Best-Fit 解码 + 两种排序策略
- **上半部分 — 核心源码逻辑**（用简洁流程图/伪代码展示）:

```
输入: 客户排列顺序 order
1. 初始化 M 架无人机路线为空
2. 按 order 顺序遍历每个客户 ci:
   a. 用 Best-Fit 选择无人机:
      - 遍历所有无人机，找「剩余容量最充足且能装入 ci」的那架
      - 若无架能装入 → 放入负载最轻的（超载由罚金处理）
3. 所有客户分配完成后，每架路线内按 EDD 排序（截止时间升序）
输出: routes[0..M-1]
```

- **下半部分 — 两种排序策略**:
  - **最近优先**（`solve_greedy`）: 客户按距仓库距离升序排列 → 优先送近处客户
  - **紧急优先**（`solve_greedy_urgent`）: 客户按类型 `urgent → normal → relaxed` 排列，同类型内按距离排序
- **右侧配一个关键优化点说明卡**:
  - ✅ Best-Fit（非 First-Fit）：选剩余容量最大的，减少碎片
  - ✅ EDD 趟内排序：同路线内截止早的先送，降低延迟
  - ✅ 紧急优先变体：简单重排序即可适应不同业务场景
- **底部**: 时间复杂度 O(n²)，0.01 秒出解，作为 SA 初始解和性能下界
- **备注**: 35秒。重点：(1) Best-Fit 和 EDD 两个优化细节 (2) 两种变体的差异化策略 (3) 为后续 SA 做铺垫——"这个解将作为模拟退火的起点"

### 第8页：模拟退火 — 原理深度讲解（成员C讲）

- **标题**: 模拟退火：从金属退火到路径优化
- **上半部分 — 物理类比（左侧配退火示意图标）**:

| 物理退火 | → | 算法映射 |
|----------|-----|----------|
| 加热金属至高温 | → | 初始温度 T₀ = 1000 |
| 原子剧烈运动（广搜） | → | 以概率接受差解 |
| 缓慢降温 | → | T ← α × T（α=0.98） |
| 最终形成低能晶格 | → | 收敛到全局最优 |

- **中间 — Metropolis 接受准则**（加框公式）:
  ```
  若 ΔE ≤ 0（新解更优）→ 必定接受
  若 ΔE > 0（新解更差）→ 以概率 exp(-ΔE / T) 接受
  ```
  **关键洞察**：温度 T 高时，exp(-ΔE/T) ≈ 1 → 几乎全接受，广泛探索；T 低时，exp(-ΔE/T) ≈ 0 → 只接受更优解，精细打磨

- **下半部分 — 5种邻域操作**（小卡片并列）:
  1. **2-opt**: 同一路线内逆转一段子路径
  2. **Swap**: 同一路线内交换两个客户
  3. **Relocate**: 将一个客户从A路线移到B路线
  4. **Exchange**: 交换AB两条路线各一个客户
  5. **Or-opt**: 将连续2-3个客户整体迁移到另一路线

- **底部一行**: 自适应冷却 — 连续50步无改善时 α ← α × 0.8，加速跳出无效搜索区域
- **备注**: 45秒，由成员C讲。核心叙事：(1) 物理直觉→算法映射 (2) 为什么接受差解不是bug而是feature——跳出局部最优 (3) 5种邻域操作各司其职

### 第9页：遗传算法

- **标题**: 遗传算法：排列编码 + 种群进化
- **要点**:
  - **编码**: 客户排列（如 `[5,1,8,3,7,2,6,4]`），解码时贪心分割为多条路线
  - **选择**: 锦标赛选择（size=3）→ 优胜劣汰
  - **交叉**: 有序交叉 OX（概率 0.8）→ 保留父代相对顺序
  - **变异**: 逆转变异 + 交换变异（概率各 0.05）→ 维持种群多样性
  - **精英保留**: 每代保留最优 5 个个体 + 早停（50代无改善）
  - **配置**: 种群 100，迭代 200 代
- **底部配图**: 建议放一张编码→解码的流程简图
- **备注**: 30秒，由成员B讲。编码方案是答辩关键——"为什么用排列编码？天然保证每个客户只访问一次"

## 六、宣讲衔接话术建议

## 七、答辩高频问题预判

| 问题 | 建议回答 |
|------|----------|
| "为什么SA比GA好？" | SA的随机重启机制更容易跳出局部最优；GA种群可能过早收敛。但GA更适合扩展到更大规模和并行化 |
| "编码为什么用排列？" | 排列天然保证每个客户只访问一次，解码时自动分段处理约束，简洁高效 |
| "Best-Fit比First-Fit好在哪？" | First-Fit可能把大需求客户塞进只剩少量容量的无人机，导致后续客户无处可放；Best-Fit选剩余容量最充足的，减少碎片化 |
| "Metropolis准则为什么有效？" | 在高温时exp(-ΔE/T)≈1，几乎接受所有差解→全局探索；低温时≈0，只接受更优解→局部精调。这正是SA能跳出局部最优的关键 |
| "成本系数怎么定的？" | 参考行业标准——电费约0.5元/km、运维0.3元/km合计0.8；延迟罚金20元/min参考外卖平台超时罚则，明显高于运营成本（1元/min），引导算法优先满足时效 |
| "怎么保证解满足约束？" | 解码阶段逐客户检查载重和航程，超限则新开一趟；目标函数含高额罚金（500元/kg, 300元/km）自动引导搜索远离不可行区域 |
| "创新点在哪？" | (1) 统一人民币成本模型让多目标优化可解释 (2) 客户类型+错峰时间窗设计有真实调度意义 (3) 四种算法在同一编码框架下系统对比+统计检验 |
| "实际能用吗？" | 算法框架可直接迁移——换真实坐标和路网数据即可。当前仿真简化了天气/地形/禁飞区，在"展望"中已讨论 |
