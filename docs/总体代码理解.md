# 无人机配送路径规划 — 总体代码理解

面向程序员的项目代码快速理解文档。重点解释核心算法与数据流。

---

## 一、项目概览

**问题**：M 架无人机从配送中心出发，为 N 个客户配送包裹。目标是最小化统一人民币总成本，受载重上限、最大航程和时间窗约束。

**四种算法**：贪心（构造基线）→ 模拟退火（元启发式）→ 遗传算法（元启发式）→ 随机搜索（随机基线）。

**代码量**：约 3000 行 Python（20+ 个源文件）。

**核心改进（v3.0）**：统一人民币成本模型 + 客户类型错峰时间窗 + 多场景对比实验 + 统计显著性检验。

## 三、核心数据流

```
1. 数据生成
   config.py → generate_data.py → CSV + dict列表

2. 问题建模
   客户dict → Customer dataclass → Problem(customers, drones, dist_matrix)

3. 算法求解
   Problem → BaseAlgorithm.solve() → (routes, cost, history)

4. 解评估
   routes → Problem.evaluate_solution() → {
       total_cost, flight_cost, operation_cost,
       delay_penalty, overload_penalty, excess_range_penalty,
       total_distance, makespan, load_gini, is_feasible, ...

   routes → Problem.get_violation_report() → 约束违反明细（硬约束+软约束）
   routes → Problem.print_violation_report() → 格式化打印违反分析

5. 可视化
   Problem + routes → visualization/*.py → PNG图表

6. 实验
   experiments/run_experiments.py → 批量运行 → JSON结果 → 统计检验
```

## 四、成本模型（统一人民币）

```python
TotalCost = 0.8 × TotalDistance(km)          # 飞行成本（电费/维护/折旧）
          + 1.0 × TotalTime(min)             # 运营时间成本（飞行+等待+服务）
          + 20.0 × TotalDelay(min)           # 延迟罚金（超时间窗末尾）
          + 500.0 × TotalOverload(kg)        # 超重罚金
          + 300.0 × TotalExcessRange(km)     # 超航程罚金
```

时间链计算：
```python
arrival = 飞到客户点的时间
wait = max(0, e - arrival)      # 提前到达→等待（不计罚金，计时间成本）
delay = max(0, arrival - l)     # 延迟到达→产生罚金
start = max(arrival, e)         # 开始服务
depart = start + service_time   # 离开
```

## 五、关键设计决策

1. **单趟模式**：每架无人机只执行一趟任务，数据生成时校验 `sum(demand) ≤ 0.85 × M × C`
2. **时间窗软约束**：允许违反但通过高额罚金引导搜索远离不可行区域
3. **客户类型错峰**：紧急(08:00) → 普通(09:00) → 宽松(10:00)，使时间窗具有调度意义
4. **排列编码+贪心解码**：GA/SA/Random均使用排列编码，解码时按载重约束贪心分割为路线
5. **道路曲折系数 β=1.3**：实际飞行距离按欧氏距离×1.3计算

## 六、算法详解

### 6.1 贪心算法 (`src/algorithms/greedy.py`)

**策略**：最近客户优先 (Nearest-First)，将所有客户按距仓库距离排序，然后贪心分配给最优无人机。

**解码器 `_decode_greedy`**（Best-Fit 策略）：
```
for each customer in order:
    best_drone = 剩余容量最充足且能装入的无人机
    if found: 分配至 best_drone
    else:     放入负载最轻的无人机（超载由罚金处理）
按 EDD(最早截止时间) 对每条路线内排序
```

**两种变体**：
| 变体 | 排序依据 | 适用场景 |
|------|----------|----------|
| `greedy` | 距仓库距离（最近优先） | 标准基线 |
| `greedy_urgent` | 客户类型(紧急→普通→宽松) + 距仓库距离 | 时间窗敏感场景 |

**复杂度**：O(N log N + N·M)，M 为无人机数，N 为客户数。

---

### 6.2 模拟退火 (`src/algorithms/sa.py`)

**初始解**：贪心算法构造。

**5 种邻域操作**：
| 操作 | 类型 | 描述 |
|------|------|------|
| `_neighbor_2opt` | 内部 | 逆转一条路线的一个子段 |
| `_neighbor_swap` | 内部 | 交换一条路线内两个客户 |
| `_neighbor_relocate` | 跨机 | 将一个客户从无人机 A 移到 B |
| `_neighbor_exchange` | 跨机 | 交换两架无人机各一个客户 |
| `_neighbor_oropt` | 跨机 | 将 2-3 个连续客户移到另一架 |

**核心循环**：
```
current = 贪心初始解
while T > T_min:
    for iter in range(iter_per_T):
        neighbor = 随机选择邻域操作(current)
        Δ = cost(neighbor) - cost(current)
        if Δ < 0 or random() < exp(-Δ/T):
            accept neighbor
            update best if improved
    // 自适应冷却
    if 连续未改善 >= patience:  T *= α × 0.8
    else:                       T *= α
```

**关键参数**（`config.py`）：
- `T0=1000, α=0.98, T_min=0.01, iter_per_T=100, patience=50`

**复杂度**：O(K × M × N²)，K 为总迭代次数（约温度层数 × 100）。

---

### 6.3 遗传算法 (`src/algorithms/ga.py`)

**编码**：排列编码（Permutation Encoding）。个体是 0..N-1 的一个排列。

**解码器 `_decode_permutation`**（贪心分割）：
```
routes = [[] for _ in range(M)]
drone_idx = 0
for ci in permutation:
    if 加入当前无人机超载: drone_idx += 1
    将 ci 加入 routes[drone_idx]
过滤空路线并返回
```

**DEAP 算子**：
| 算子 | 实现 |
|------|------|
| 选择 | 锦标赛选择 (tournsize=3) |
| 交叉 | Ordered Crossover (`cxOrdered`) |
| 变异 | Shuffle Indexes (`mutShuffleIndexes`, indpb=0.05) |
| 精英保留 | 每代保留 elite_size=5 个最优个体 |

**核心循环**：
```
pop = 随机排列种群(PopSize=100)
for gen in range(1, N_Gen=200):
    选择 + 精英保留
    按概率交叉(cx=0.8) + 变异(mut=0.15)
    评估新个体
    HallOfFame 更新
    早停检查(连续50代无改善则终止)
解码 HallOfFame[0] → routes
```

**复杂度**：O(G × P × M × N)，G=代数，P=种群规模。

---

### 6.4 随机搜索 (`src/algorithms/random_search.py`)

**策略**：生成 N=10000 个随机排列，用与 GA 相同的 `_decode_permutation` 解码器评估，取最优。

**目的**：验证元启发式算法的有效性——如果 SA/GA 不比随机搜索好，说明算法无效。

**复杂度**：O(N_iter × M × N)。

---

### 6.5 算法对比总结

| 算法 | 类型 | 编码方式 | 解码器 | 搜索能力 | 速度 |
|------|------|----------|--------|----------|------|
| 贪心 | 构造基线 | 距离排序 | Best-Fit | 局部 | 极快 |
| 模拟退火 | 元启发式 | 路线直接操作 | 直接修改 | 全局 | 中等 |
| 遗传算法 | 元启发式 | 排列编码 | 贪心分割 | 全局 | 较慢 |
| 随机搜索 | 随机基线 | 排列编码 | 贪心分割 | 全局(随机) | 慢 |

**关键设计**：GA 和 Random Search 共享同一个排列→路线解码器，因此它们的性能差异完全归因于搜索策略，而非解码方式。SA 直接操作路线而非排列，搜索粒度更细。

## 七、数据模型

### 7.1 Customer (`src/models/customer.py`)

```python
@dataclass
class Customer:
    id: int                         # 0-based 索引
    x, y: float                     # 坐标 (km)
    demand: float                   # 需求量 (kg)
    customer_type: str              # "urgent" | "normal" | "relaxed"
    time_window: tuple[float, float]  # (最早, 最晚) 小时
    service_time: float             # 服务时间 (小时)
```

### 7.2 Drone (`src/models/drone.py`)

```python
@dataclass
class Drone:
    id: int
    capacity: float       # 载重上限 (kg), 默认 20
    speed: float          # 速度 (km/h), 默认 40
    max_range: float      # 最大航程 (km), 默认 150
```

### 7.3 Problem (`src/models/problem.py`)

核心类，封装：
- **数据**：客户列表、无人机列表、距离矩阵 `(N+1)×(N+1)`
- **时间链计算** (`compute_route_timeline`)：逐站计算 arrival/wait/delay/start/depart
- **解评估** (`evaluate_solution`)：返回统一人民币总成本 + 分解项 + 基尼系数
- **约束违反报告** (`get_violation_report` / `print_violation_report`)：区分硬约束（载重、航程、去重、出发返回）和软约束（时间窗延迟/等待）
- **基尼系数** (`_gini`)：衡量负载均衡度

### 7.4 数据生成 (`data/generate_data.py`)

- **坐标**：70% 高斯分布（热区）+ 30% 均匀分布（边缘）
- **客户类型**：20% urgent / 60% normal / 20% relaxed
- **时间窗**：按客户类型错峰生成，窗口宽度依类型不同
- **可行性校验**：重试机制确保 `Σ demand ≤ 0.85 × M × C`
- **输出**：CSV 文件到 `data/output/`

## 十二、扩展点与注意事项

1. **多趟模式**：当前为单趟模式，扩展为多趟需修改贪心解码器支持无人机返回后重新出发
2. **动态需求**：当前为静态问题，扩展到动态需要实时重规划模块
3. **三维路径**：如需考虑建筑物避障，可将距离计算改为 3D A* 或 RRT
4. **充电约束**：如需加入充电站，需修改航程约束为能量模型
5. **DEAP 幂等性**：`_create_toolbox` 每次被调用时检查 `creator.__dict__` 避免重复注册（Streamlit 热重载需要此保护）
