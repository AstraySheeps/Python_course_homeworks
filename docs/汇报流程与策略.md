# 无人机集群配送路径规划 — 汇报流程与策略

## 一、汇报形式

- 时间：5分钟汇报 + 4分钟问答（每组约9分钟）
- 人员：全组3-4人，每人需承担汇报内容
- 形式：PPT演讲 + 可选Demo演示

---

## 二、汇报时间分配（5分钟）

| 时间 | 内容 | 负责人 |
|------|------|--------|
| 0:00-0:30 | 项目背景与问题定义 | 成员A |
| 0:30-1:00 | 数据描述与预处理 | 成员A |
| 1:00-2:30 | 算法设计与实现（核心） | 成员B |
| 2:30-3:30 | 实验结果与对比分析 | 成员C |
| 3:30-4:30 | 可视化展示与案例分析 | 成员C |
| 4:30-5:00 | 总结与展望 | 成员D（或组长） |

如果4人组，可将"算法设计"拆为"遗传算法"（成员B）和"模拟退火"（成员C），"实验结果"由成员D讲。

---

## 三、各环节汇报要点

### Part 1：项目背景（30秒）

**必须说清楚**：
- 低空物流是AI优化算法的典型应用场景
- 核心问题：多无人机 + 多客户点 → 最短总路径
- 本质是带容量约束的车辆路径问题（CVRP）的变体

**PPT上放**：一张问题示意图（配送中心 + 客户点 + 无人机）

### Part 2：数据与预处理（30秒）

**必须说清楚**：
- 仿真数据参数（30客户/10/50三场景、坐标范围[0,50]、重量1~8kg自适应、配送中心(25,25)）
- 约束条件（载重≤20kg、单趟航程≤150km、时间窗软约束）
- 数据清洗流程（一句话带过）

**PPT上放**：客户点分布散点图 + 约束条件卡片

### Part 3：算法设计（90秒，重中之重）

这是核心部分，要讲清楚但不陷入细节：

**贪心算法**（15秒）：
- "基线方法，每次选最近的可配送客户"
- PPT上放：贪心路径图 + 总距离

**遗传算法**（40秒）：
- 编码方式：客户排列编码（说清楚"为什么用排列而不是0-1编码"）
- 解码过程：按排列顺序贪心打包成路线（边说边放流程图）
- 遗传算子：锦标赛选择 + 有序交叉 + 索引洗牌变异 + 精英保留
- PPT上放：编码/解码示意图 + 进化曲线

**模拟退火**（35秒）：
- 类比："像金属退火，高温时大范围搜索，降温后局部精调"
- 邻域算子：交换、反转、插入（放动画或示意图）
- Metropolis准则：更优的必定接受，更差的以一定概率接受
- PPT上放：退火搜索轨迹图 + 重启收敛对比图

### Part 4：实验结果（60秒）

**必须展示的数据**：
- 三算法对比表（总距离、趟次、相比贪心提升百分比）—— 核心
- 进化曲线（遗传算法的收敛过程）
- 退火轨迹（多次重启的搜索过程）

**要点**：
- 强调"模拟退火结果最优，遗传算法接近，两者均大幅优于贪心"
- 提一句运行时间（如果有的话）

### Part 5：可视化展示（60秒）

- 三算法路径对比图（并排展示，一目了然）
- 单架无人机详情图（选最有代表性的1-2架）
- 可选：放一段 Streamlit Demo 录屏（10-15秒）

### Part 6：总结与展望（30秒）

- 一句话总结："本项目的三种算法均实现了对贪心基线的大幅改进"
- 未来工作：时间窗约束、多配送中心、强化学习方案
- 致谢

## 四、4分钟问答准备（预判高频问题）

| 可能被问到的问题 | 准备答案 |
|-----------------|---------|
| "为什么遗传算法的解不如模拟退火？" | 遗传算法种群收敛到局部最优，模拟退火的随机重启机制更有优势跳出局部最优 |
| "种群规模为什么选200？" | 我们做了网格搜索（100/200/300），200在效果和计算开销间平衡最优 |
| "如何保证解满足约束？" | 解码时逐客户检查载重和里程，不满足则新开一趟；超限客户加惩罚项 |
| "编码为什么用排列而不是直接编码路线？" | 排列编码天然保证每个客户只访问一次，解码时自动处理约束，简单高效 |
| "如果客户数增加到100个呢？" | 遗传/退火仍适用但需增加迭代次数；贪心算法扩展性最好但解质量差 |
| "实际场景和仿真的差距？" | 仿真简化了天气、地形、禁飞区等因素，但算法框架可直接扩展 |
| "你们的创新点在哪？" | 将三种算法统一到同一编码框架下系统对比，超参数经过网格搜索调优 |

## 五、演练建议

1. **计时练习**：至少完整演练3遍，严格控制5分钟
2. **设备检查**：提前到教室测试投影、分辨率、字体
3. **分工衔接**：每人讲完要自然过渡到下一个人（"接下来由XXX介绍算法实现"）
4. **备用方案**：准备PDF版PPT（防止PPTX版本兼容问题）
5. **眼神交流**：不要全程念稿，面向观众而非屏幕
